Mermaid Sequence Diagram Syntax Guide

This comprehensive guide covers the syntax for creating sequence diagrams with Mermaid.js.

1. Participants & Actors

Define participants explicitly to control their order. You can use aliases and different shapes (actor vs participant).

sequenceDiagram
    actor A as Alice
    participant B as Bob
    participant C as Carol
    A->>B: Hello Bob
    B->>C: Hello Carol
sequenceDiagram actor A as Alice participant B as Bob participant C as Carol A->>B: Hello Bob B->>C: Hello Carol

2. Messages

Different arrow types represent different types of messages (synchronous, asynchronous, return, etc.).

sequenceDiagram
    A->B: Solid line (no arrow)
    A-->B: Dotted line (no arrow)
    A->>B: Solid line (arrowhead)
    A-->>B: Dotted line (arrowhead)
    A-xB: Solid line (cross)
    A--xB: Dotted line (cross)
    A-)B: Solid line (open arrow)
    A--)B: Dotted line (open arrow)
sequenceDiagram participant A participant B A->B: Solid line (no arrow) A-->B: Dotted line (no arrow) A->>B: Solid line (arrowhead) A-->>B: Dotted line (arrowhead) A-xB: Solid line (cross) A--xB: Dotted line (cross) A-)B: Solid line (open arrow) A--)B: Dotted line (open arrow)

3. Activations

Show when a participant is active/processing. Use activate/deactivate or the +/- shortcut.

sequenceDiagram
    Alice->>+Bob: Request
    Bob->>+Carol: Forward
    Carol-->>-Bob: Response
    Bob-->>-Alice: Final Response
sequenceDiagram Alice->>+Bob: Request Bob->>+Carol: Forward Carol-->>-Bob: Response Bob-->>-Alice: Final Response

4. Notes

Add notes relative to participants: right of, left of, or over.

sequenceDiagram
    participant A
    participant B
    Note left of A: Left Note
    Note right of B: Right Note
    Note over A,B: Spanning Note
sequenceDiagram participant A participant B Note left of A: Left Note Note right of B: Right Note Note over A,B: Spanning Note

5. Loops

Indicate repeating sequences using loop [Label] ... end.

sequenceDiagram
    Alice->>Bob: Start
    loop Every minute
        Bob->>Bob: Check status
    end
sequenceDiagram Alice->>Bob: Start loop Every minute Bob->>Bob: Check status end

6. Alt / Opt (Logic)

Use alt for if/else logic and opt for optional sequences.

sequenceDiagram
    alt Success
        A->>B: 200 OK
    else Error
        A->>B: 500 Error
    end
    opt Extra
        A->>B: Optional Log
    end
sequenceDiagram alt Success A->>B: 200 OK else Error A->>B: 500 Error end opt Extra A->>B: Optional Log end

7. Parallel Actions (Par)

Show actions happening at the same time using par [Label] ... and [Label] ... end.

sequenceDiagram
    par Parallel Action
        A->>B: Task 1
    and
        A->>C: Task 2
    end
    B-->>A: Done 1
    C-->>A: Done 2
sequenceDiagram par Parallel Action A->>B: Task 1 and A->>C: Task 2 end B-->>A: Done 1 C-->>A: Done 2

8. Critical Region

Highlight critical sections that must not be interrupted using critical [Label] ... option [Label] ... end.

sequenceDiagram
    critical Database Transaction
        A->>DB: Update
    option Network Failure
        A->>A: Retry
    end
sequenceDiagram critical Database Transaction A->>DB: Update option Network Failure A->>A: Retry end

9. Break

Indicate a stop in the sequence (like an exception or return) using break [Label] ... end.

sequenceDiagram
    A->>B: Request
    break Invalid Input
        B-->>A: Error Response
    end
    B-->>A: Success Response
sequenceDiagram A->>B: Request break Invalid Input B-->>A: Error Response end B-->>A: Success Response

10. Background Highlight (Rect)

Color background areas using rect [rgb/color] ... end.

sequenceDiagram
    rect rgb(200, 255, 200)
        A->>B: Success Path
    end
    rect rgb(255, 200, 200)
        A->>B: Error Path
    end
sequenceDiagram rect rgb(200, 255, 200) A->>B: Success Path end rect rgb(255, 200, 200) A->>B: Error Path end

11. Autonumber

Automatically number the messages using autonumber.

sequenceDiagram
    autonumber
    A->>B: Step 1
    B->>C: Step 2
    C-->>B: Step 3
sequenceDiagram autonumber A->>B: Step 1 B->>C: Step 2 C-->>B: Step 3

12. Comments

Add comments that are ignored by the parser using %%.

sequenceDiagram
    %% This is a comment
    A->>B: Hello
    %% Another comment
    B-->>A: Hi
sequenceDiagram %% This is a comment A->>B: Hello %% Another comment B-->>A: Hi
